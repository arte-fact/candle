The Q5_K chunked `_cuda8` that was #1 at 3280 ms has **disappeared
from the top 15**. Per-call: 820 / 24 = **34 ms/call** (single-launch
kernel) vs pre-P5's 24 × 144 × 956 µs = 3305 ms (3456 chunked launches
at 956 µs each). That's ~**144× fewer launches**, with each launch
only ~36× slower than a per-chunk — a big net win.

## Why per-call Q5_K MMQ is 34 ms vs Q4_1's 18 ms

The Q5_K kernel does **~8× more sub-block work per super-block**: each
256-element super-block has 8 sub-blocks with per-sub-block scale +
min unpacking, vs Q4_1's single 32-element block. Per thread, for a
4096-element K dim: Q4_1 iterates 128 32-elem blocks × 4 dp4a = 512
dp4a per col. Q5_K iterates 16 256-elem super-blocks × 8 sub-blocks ×
8 dp4a (for sumi_d) + 8 dp4a (for sumi_y) = 2048 dp4a per col — 4×
more. The ~2× ratio (34 / 18) is better than the 4× work multiplier
because Q5_K's ql/qh data has much better spatial locality (one 32-byte
qs read per 2 sub-blocks) so L1 bandwidth amortises well.

## Priority shuffle

1. **P4** — fused norm+silu+quantize. Now the biggest disproportionate
   category (913 ms vs turbo's 134 ms = 6.8×). Simple port, ~750 ms
   expected savings.
2. **P1 Phase 2d** — Q4_1 MMQ LDS staging. Bigger absolute win (2.7 s)
   but 5× the engineering effort of P4. Second priority.
3. **Decode parity** — decode is now at 36 t/s vs turbo 58.76, which
   is a real 1.6× gap. But nothing in the top 15 suggests a cheap
   fix — the decode hot path is `mul_mat_vec_q4_1_q8_1_cuda1`
   (1171 ms over 16k calls × 72 µs each). Each call is already as
   fast as a single-warp kernel can be. The only decode lever is
   further kernel fusion (P4) and maybe prefetching, but the ceiling
   is bounded.

## How to reproduce

```bash
cd /artefact/candle
# Rebuild:
cargo build -p candle-examples --features hip --release --example quantized-qwen35
# Triple-run (non-profiled):
PROMPT="$(cat /tmp/long_prompt.txt)"
for i in 1 2 3; do
    ./target/release/examples/quantized-qwen35 \
        --model /artefact/models/Qwen3.5-9B-Q4_1.gguf \
        --prompt "$PROMPT" --sample-len 128 --temperature 0 \
        2>/dev/null | grep -E "prompt tokens|tokens generated" | tail -2
done
# Profiled (needs rocprofv3):
rocprofv3 --kernel-trace -f csv -d /tmp/bench-p5 -o p5 -- \
    ./target/release/examples/quantized-qwen35 \
    --model /artefact/models/Qwen3.5-9B-Q4_1.gguf \
    --prompt "$PROMPT" --sample-len 128 --temperature 0
python3 /tmp/bench-p0/analyze.py /tmp/bench-p5/p5_kernel_trace.csv
```
